# 提取并整理功能说明

## 功能概述
在文件列表界面的右上角菜单中新增了"提取并整理"功能，可以一键完成以下操作：

1. **递归提取文件**：从当前目录下的所有子文件夹（包括嵌套的子文件夹）中提取所有文件到当前目录
2. **自动归类整理**：按文件类型自动将文件移动到对应的分类文件夹
3. **清理空文件夹**：删除提取后留下的空文件夹

## 使用方法

1. 进入需要整理的目录
2. 点击右上角的"更多"菜单（三个点图标）
3. 选择"提取并整理"选项
4. 确认操作提示后，系统将自动完成整理

## 文件分类规则

文件将按以下规则自动归类到对应文件夹：

- **图片**：图片类型文件（jpg, png, gif, webp 等）
- **视频**：视频类型文件（mp4, avi, mkv, mov 等）
- **音频**：音频类型文件（mp3, flac, wav, aac 等）
- **文档**：文档类型文件（word, excel, ppt, pdf, txt 等）

其他类型的文件将保留在当前目录。

## 操作流程示例

假设当前目录结构如下：
```
/我的文件/
├── 文件夹A/
│   ├── 照片1.jpg
│   └── 视频1.mp4
├── 文件夹B/
│   ├── 子文件夹/
│   │   └── 音乐1.mp3
│   └── 文档1.pdf
└── 文件1.txt
```

执行"提取并整理"后：
```
/我的文件/
├── 图片/
│   └── 照片1.jpg
├── 视频/
│   └── 视频1.mp4
├── 音频/
│   └── 音乐1.mp3
└── 文档/
    ├── 文档1.pdf
    └── 文件1.txt
```

## 注意事项

⚠️ **重要提示**：
- 此操作不可撤销，请谨慎使用
- 建议在执行前确认当前目录下的文件结构
- 如果存在同名文件，可能会导致移动失败
- 操作完成后会显示详细的执行结果统计

## 技术实现

### 菜单项添加
在 `lib/screen/file_list/file_list_menu_anchor.dart` 中：
- 添加了 `MenuId.extractAndOrganize` 枚举值
- 在 `menuGroupOperations` 中添加了新的菜单项

### 功能实现
在 `lib/screen/file_list/file_list_screen.dart` 中：
- `_extractAndOrganize()`: 显示确认对话框
- `_doExtractAndOrganize()`: 执行主要逻辑
- `_collectFilesRecursively()`: 递归收集所有子文件夹中的文件

### 执行步骤
1. 递归遍历所有子文件夹，收集文件列表和空文件夹列表
2. 将所有文件移动到当前目录
3. 刷新文件列表
4. 按类型归类文件到对应文件夹
5. 删除空文件夹
6. 显示操作结果统计

## UI 优化
- 对话框使用圆角设计（20px）
- 按钮样式统一，取消按钮使用次要颜色
- 确认按钮使用 FilledButton 样式
- 加载提示不可取消，防止操作中断
- 结果提示显示 3 秒，便于查看详细信息
