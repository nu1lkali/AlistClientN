# 提取并整理功能说明

## 功能概述
在文件列表界面的右上角菜单中新增了"提取并整理"功能，可以一键完成以下操作：

1. **递归提取文件**：从当前目录下的所有子文件夹（包括嵌套的子文件夹）中提取所有文件到当前目录
2. **自动归类整理**：按文件类型自动将文件移动到对应的分类文件夹
3. **清理空文件夹**：删除提取后留下的空文件夹

## 使用方法

1. 进入需要整理的目录
2. 点击右上角的"更多"菜单（三个点图标）
3. 选择"提取并整理"选项
4. 确认操作提示后，系统将自动完成整理

## 文件分类规则

文件将按以下规则自动归类到对应文件夹：

- **图片**：图片类型文件（jpg, png, gif, webp 等）
- **视频**：视频类型文件（mp4, avi, mkv, mov 等）
- **音频**：音频类型文件（mp3, flac, wav, aac 等）
- **文档**：文档类型文件（word, excel, ppt, pdf, txt 等）

其他类型的文件将保留在当前目录。

## 操作流程示例

### 示例 1：基本提取和整理

假设当前目录结构如下：
```
/我的文件/
├── 文件夹A/
│   ├── 照片1.jpg
│   └── 视频1.mp4
├── 文件夹B/
│   ├── 子文件夹/
│   │   └── 音乐1.mp3
│   └── 文档1.pdf
└── 文件1.txt
```

执行"提取并整理"后：
```
/我的文件/
├── 图片/
│   └── 照片1.jpg
├── 视频/
│   └── 视频1.mp4
├── 音频/
│   └── 音乐1.mp3
└── 文档/
    ├── 文档1.pdf
    └── 文件1.txt
```

### 示例 2：同名文件自动重命名

假设当前目录结构如下：
```
/我的文件/
├── 照片.jpg (已存在)
├── 文档.pdf (已存在)
├── 文件夹A/
│   ├── 照片.jpg (同名)
│   └── 视频.mp4
└── 文件夹B/
    ├── 照片.jpg (同名)
    └── 文档.pdf (同名)
```

执行"提取并整理"后：
```
/我的文件/
├── 图片/
│   ├── 照片.jpg (原有)
│   ├── 照片(1).jpg (从文件夹A提取并重命名)
│   └── 照片(2).jpg (从文件夹B提取并重命名)
├── 视频/
│   └── 视频.mp4
└── 文档/
    ├── 文档.pdf (原有)
    └── 文档(1).pdf (从文件夹B提取并重命名)
```

操作结果提示：
```
提取文件：3 个（重命名 3 个）
归类整理：5 个
删除空文件夹：2 个
```

## 注意事项

⚠️ **重要提示**：
- 此操作不可撤销，请谨慎使用
- 建议在执行前确认当前目录下的文件结构
- **同名文件自动重命名**：如果提取的文件与当前目录中的文件同名，会自动添加序号重命名（例如：`文件.txt` → `文件(1).txt`）
- 操作完成后会显示详细的执行结果统计，包括重命名的文件数量

## 技术实现

### 菜单项添加
在 `lib/screen/file_list/file_list_menu_anchor.dart` 中：
- 添加了 `MenuId.extractAndOrganize` 枚举值
- 在 `menuGroupOperations` 中添加了新的菜单项

### 功能实现
在 `lib/screen/file_list/file_list_screen.dart` 中：
- `_extractAndOrganize()`: 显示确认对话框
- `_doExtractAndOrganize()`: 执行主要逻辑
- `_collectFilesRecursively()`: 递归收集所有子文件夹中的文件

### 执行步骤
1. 递归遍历所有子文件夹，收集文件列表和空文件夹列表
2. 检查文件名冲突，对同名文件自动重命名（添加序号）
3. 将所有文件移动到当前目录
4. 刷新文件列表
5. 按类型归类文件到对应文件夹
6. 删除空文件夹
7. 显示操作结果统计（包括重命名数量）

### 同名文件处理
当提取的文件与目标目录中的文件同名时，系统会自动重命名：
- 原文件名：`照片.jpg`
- 重命名后：`照片(1).jpg`
- 如果 `照片(1).jpg` 也存在，则命名为 `照片(2).jpg`，以此类推

重命名规则：
- 保留原文件扩展名
- 在文件名后添加括号和序号
- 自动递增序号直到找到不冲突的名称
