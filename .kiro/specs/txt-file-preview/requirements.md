# 需求文档

## 简介

本功能为Alist Flutter客户端添加txt文件的内置预览功能。当前应用已支持图片、视频、PDF等文件的预览，但txt文件只能通过下载后使用外部应用打开。本功能将提供一个专用的txt文件预览界面，支持直接在应用内查看文本内容，提升用户体验。

## 术语表

- **TxtPreviewScreen**: txt文件预览界面组件
- **FileListScreen**: 文件列表界面，用户点击文件的入口
- **FileReaderScreen**: 当前用于处理无法直接预览的文件类型的界面（下载后用外部应用打开）
- **AlistClient**: Alist服务器的Flutter客户端应用
- **RemotePath**: 文件在Alist服务器上的路径
- **Sign**: 文件访问签名，用于授权访问

## 需求

### 需求 1: 打开txt文件预览

**用户故事:** 作为用户，我想要在应用内直接查看txt文件内容，这样我就不需要下载文件并使用外部应用打开。

#### 验收标准

1. WHEN 用户在文件列表中点击txt文件 THEN 系统应打开txt预览界面
2. WHEN txt预览界面打开时 THEN 系统应显示加载指示器
3. WHEN txt文件内容加载完成 THEN 系统应在预览界面中显示文本内容
4. WHEN txt文件加载失败 THEN 系统应显示错误信息并提供重试选项

### 需求 2: 显示文本内容

**用户故事:** 作为用户，我想要清晰地阅读txt文件内容，这样我就能快速获取文件信息。

#### 验收标准

1. THE 系统应以可读的字体和字号显示文本内容
2. THE 系统应支持文本内容的垂直滚动
3. WHEN 文本内容超过屏幕高度 THEN 系统应提供流畅的滚动体验
4. THE 系统应正确处理不同的文本编码（UTF-8、GBK等）
5. THE 系统应保留文本的原始格式（换行、空格、缩进）

### 需求 3: 文本编码处理

**用户故事:** 作为用户，我想要系统自动识别文本编码，这样我就能正确查看不同编码的txt文件。

#### 验收标准

1. WHEN 系统加载txt文件 THEN 系统应尝试自动检测文本编码
2. WHEN 自动检测失败或显示乱码 THEN 系统应提供手动选择编码的选项
3. THE 系统应至少支持UTF-8、GBK、GB2312编码
4. WHEN 用户切换编码 THEN 系统应重新加载并显示文本内容

### 需求 4: 界面导航和控制

**用户故事:** 作为用户，我想要方便地控制预览界面，这样我就能轻松返回或执行其他操作。

#### 验收标准

1. THE 预览界面应在顶部显示应用栏（AppBar）
2. THE 应用栏应显示文件名
3. THE 应用栏应包含返回按钮
4. WHEN 用户点击返回按钮 THEN 系统应关闭预览界面并返回文件列表
5. THE 应用栏应包含更多操作菜单按钮

### 需求 5: 文件操作功能

**用户故事:** 作为用户，我想要在预览界面执行常用文件操作，这样我就不需要返回文件列表。

#### 验收标准

1. WHEN 用户点击更多操作菜单 THEN 系统应显示操作选项列表
2. THE 操作选项应包含"复制链接"功能
3. THE 操作选项应包含"下载文件"功能
4. THE 操作选项应包含"分享"功能
5. WHEN 用户选择"复制链接" THEN 系统应将文件链接复制到剪贴板并显示提示
6. WHEN 用户选择"下载文件" THEN 系统应将文件添加到下载队列并显示提示
7. WHEN 用户选择"分享" THEN 系统应打开系统分享对话框

### 需求 6: 大文件处理

**用户故事:** 作为用户，我想要系统能够处理大型txt文件，这样我就能查看各种大小的文本文件。

#### 验收标准

1. WHEN txt文件大小超过10MB THEN 系统应显示警告对话框
2. THE 警告对话框应提示用户文件较大，可能影响性能
3. THE 警告对话框应提供"继续预览"和"取消"选项
4. WHEN 用户选择"继续预览" THEN 系统应加载并显示文件内容
5. WHEN 用户选择"取消" THEN 系统应返回文件列表
6. WHEN 文件大小超过50MB THEN 系统应拒绝预览并建议下载

### 需求 7: 文本搜索功能

**用户故事:** 作为用户，我想要在txt文件中搜索特定文本，这样我就能快速定位到需要的内容。

#### 验收标准

1. THE 应用栏应包含搜索按钮
2. WHEN 用户点击搜索按钮 THEN 系统应显示搜索输入框
3. WHEN 用户输入搜索关键词 THEN 系统应高亮显示所有匹配的文本
4. THE 系统应显示匹配结果的数量
5. THE 系统应提供"上一个"和"下一个"按钮用于在匹配结果间导航
6. WHEN 用户点击"上一个"或"下一个" THEN 系统应滚动到对应的匹配位置

### 需求 8: 文本选择和复制

**用户故事:** 作为用户，我想要选择和复制文本内容，这样我就能将需要的内容复制到其他地方。

#### 验收标准

1. WHEN 用户长按文本 THEN 系统应进入文本选择模式
2. THE 系统应显示选择手柄用于调整选择范围
3. THE 系统应显示包含"复制"选项的上下文菜单
4. WHEN 用户选择"复制" THEN 系统应将选中的文本复制到剪贴板
5. WHEN 复制成功 THEN 系统应显示提示信息

### 需求 9: 性能优化

**用户故事:** 作为用户，我想要预览界面响应迅速，这样我就能获得流畅的使用体验。

#### 验收标准

1. WHEN 文件小于1MB THEN 系统应在2秒内完成加载
2. WHEN 用户滚动文本 THEN 系统应保持60fps的流畅度
3. THE 系统应使用流式加载处理大文件
4. THE 系统应缓存已加载的文件内容
5. WHEN 用户再次打开相同文件 THEN 系统应从缓存加载以提升速度

### 需求 10: 错误处理

**用户故事:** 作为系统管理员，我想要系统能够优雅地处理各种错误情况，这样用户就能获得清晰的错误信息和解决方案。

#### 验收标准

1. WHEN 网络连接失败 THEN 系统应显示"网络连接失败"错误信息
2. WHEN 文件不存在或已删除 THEN 系统应显示"文件不存在"错误信息
3. WHEN 文件访问权限不足 THEN 系统应显示"权限不足"错误信息
4. WHEN 文件格式无法识别 THEN 系统应显示"无法识别的文件格式"错误信息
5. THE 所有错误信息应提供"重试"按钮
6. WHEN 用户点击"重试" THEN 系统应重新尝试加载文件
